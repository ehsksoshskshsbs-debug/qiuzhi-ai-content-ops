# 用户反馈运营

## 处理顺序

1. 保存来源平台和内容编号。
2. 匿名化个人信息。
3. 去除完全重复内容，但保留出现次数。
4. 判断情绪：正向、中性、负向、求助。
5. 选择需求标签。
6. 评估优先级。
7. 建议处理动作和负责人角色。

## 需求标签

安装与环境、步骤看不懂、功能限制、价格与权限、模板或资料、新选题建议、纠错与事实核验、合作或商务、社群与招聘、无效或风险内容。

## 优先级

| 等级 | 条件 | 建议 |
| --- | --- | --- |
| P0 | 安全、隐私、严重事实错误或大面积不可用 | 暂停传播并立即核验 |
| P1 | 高频阻塞、明显误解或核心步骤失败 | 当日处理并更新置顶说明 |
| P2 | 个别问题、资料请求或优化建议 | 进入日常回复与选题库 |
| P3 | 低相关、重复或无需动作 | 留档或不处理 |

## 匿名化

将姓名替换为 `[用户A]`，手机号替换为 `[手机号已隐藏]`，邮箱替换为 `[邮箱已隐藏]`，微信号、地址、订单号和公司内部信息采用同样规则。

不得推测用户的真实身份、职业、健康、收入或其他敏感属性。

## 台账字段

`feedback_id`、`content_id`、`platform`、`received_at`、`anonymized_text`、`duplicate_count`、`sentiment`、`need_tag`、`priority`、`status`、`recommended_action`、`owner_role`、`is_mock`。
