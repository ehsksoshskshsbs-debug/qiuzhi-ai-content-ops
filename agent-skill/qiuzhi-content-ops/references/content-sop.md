# 内容生产与复盘 SOP

## 选题评分

每项 1—5 分，总分 30 分。23 分及以上建议进入制作；18—22 分先优化角度；17 分及以下进入观察库。

| 维度 | 判断问题 |
| --- | --- |
| 用户痛点 | 问题是否具体且高频？ |
| 结果价值 | 观众完成后能得到什么？ |
| 账号匹配 | 是否符合小白友好、真实干活？ |
| 演示可行 | 是否能用画面证明，而不是只讲概念？ |
| 差异化 | 是否有独特场景、流程或避坑？ |
| 时效与风险 | 是否仍然有效，且风险可控？ |

## 内容编号

使用 `QZ-YYYYMMDD-主题缩写-序号`，例如 `QZ-20260721-WEEKLY-01`。选题卡、稿件、日历、反馈和复盘必须使用同一编号。

## 标准流程

1. 收集趋势和用户问题。
2. 建立选题卡并评分。
3. 核验产品功能、价格、权限和发布时间。
4. 定义一个可被画面验证的核心结果。
5. 完成 B站主内容大纲。
6. 按五平台规则适配。
7. 执行事实、版权、隐私和表达审核。
8. 建立七天发布日历。
9. 发布后按内容编号收集反馈。
10. 在约定观察窗结束后复盘并更新选题库。

## 来源记录

每条动态事实保存：主张、来源标题、链接、发布或更新日期、查询日期、是否已交叉核验、备注。

## 复盘框架

按顺序输出：原目标、实际结果、与目标的差异、可观察到的用户反馈、解释假设、保留动作、停止动作、下一轮单变量实验。

不要把播放量高直接解释为某一个标题元素导致。把解释写成待验证假设。
