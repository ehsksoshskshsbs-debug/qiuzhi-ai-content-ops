# 五平台内容打法

## B站

目标：建立完整理解与收藏价值。

输出：备选标题、封面字、结果预览、章节大纲、逐段口播、屏幕演示、简介、置顶评论和评论维护问题。

结构：结果先行 → 为什么值得做 → 前置准备 → 分步演示 → 常见失败 → 适用边界 → 总结与下一步。

检查：章节是否可独立理解；演示是否写明成功信号；是否提供失败兜底；标题承诺是否在内容中兑现。

## 抖音

目标：在短时间内展示一个明确收益。

输出：三秒钩子、60 秒内口播、镜头/录屏说明、重点字幕、结尾互动。

结构：痛点或结果 → 三步演示 → 前后对比 → 行动引导。

检查：前五秒是否出现结果；每个镜头是否只有一个信息重点；是否避免把复杂教程压成无法跟做的碎片。

## 视频号

目标：让职场用户理解价值并愿意转发。

输出：场景式开场、克制口播、步骤、使用边界、转发或收藏引导。

结构：常见工作困扰 → 可复制方法 → 风险提醒 → 转发给需要的人。

检查：避免过强营销口吻；突出工作场景；保证静音观看也能理解。

## 小红书

目标：通过搜索、收藏和卡片化表达获得长期访问。

输出：痛点标题、封面文案、八页卡片、正文、关键词、评论区问题。

八页建议：结果 → 谁适合 → 准备 → 步骤1 → 步骤2 → 步骤3 → 避坑 → 总结清单。

检查：封面和标题是否具体；每页是否能单独截屏阅读；正文是否补充卡片未展开的条件；关键词是否自然。

## 公众号

目标：沉淀完整教程和可检索资料。

输出：标题、摘要、导语、分级标题、完整步骤、注意事项、常见问题、结语和资料路径。

结构：问题背景 → 最终效果 → 工具与条件 → 完整教程 → 常见错误 → 适用边界 → 总结。

检查：信息是否完整；外部链接和产品功能是否核验；图片位置是否有说明；标题层级是否清晰。

## 跨平台改写原则

- 保留同一事实底座，改变叙事顺序和用户动作。
- 不把 B站简介直接复制成小红书正文。
- 不把短视频钩子当作公众号标题。
- 每个平台保留独立的评论维护问题。
- 最新字符限制、链接规则或社区规范必须在发布前核验。
