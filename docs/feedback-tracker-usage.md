# 反馈追踪工作台使用说明

## 入口与权限

公开作品集继续对外开放，运营工作台位于 `/ops`。工作台需要 ChatGPT 登录，并只允许部署环境变量 `OPS_ALLOWED_EMAILS` 中的邮箱访问。多个邮箱使用英文逗号分隔。

成员白名单应作为部署密钥管理，不写入 Git、网页源码、数据库或截图。

## 新增反馈

点击“新增记录”，填写平台、来源类型、匿名摘要、优先级、负责人和下一步。系统会自动生成 `FB-日期-随机码` 形式的追踪编号，并记录创建人和时间。

摘要中常见的手机号、邮箱和微信号会被自动遮盖，但录入前仍应先进行人工脱敏。不要粘贴私信原文、客户秘密、公司内部数据或不必要的真实昵称。

## CSV 导入

在工作台下载模板，保留以下表头：

```text
平台,来源类型,平台记录编号,内容编号,用户代号,匿名摘要,情绪,需求标签,优先级,状态,负责人,下一步,发生时间
```

单次最多导入 500 条。存在“平台＋平台记录编号”重复时会跳过，其他错误会返回原 CSV 行号和安全错误说明。

## 可追踪历史

点击任意台账记录可查看和编辑。状态、负责人、优先级、摘要和下一步等关键变化都会产生事件记录，包括操作人和时间。首版删除采用“已归档”，不做不可恢复的硬删除。

## 运营指标

运营数据区可以按平台、内容编号和日期保存指标快照，例如播放量、收藏量、评论量或关注转化。相同平台、内容、日期和指标名再次保存时会更新原快照。

## 平台自动同步

B站、抖音、小红书、视频号和公众号均有独立连接状态。没有官方开发者权限和安全授权时显示“待接入”，点击检查授权只记录一次安全的检查结果，不会抓取、回复或发布内容。

启用真实同步时，需要逐个平台提供官方 API 权限，并把令牌保存在部署密钥中。令牌不得写入 D1 或提交到仓库。

## 数据导出

“导出 CSV”会按当前筛选条件生成文件，默认不包含外部用户真实标识。导出文件不会长期保存在云端，下载后应按内部数据规范保存和清理。
