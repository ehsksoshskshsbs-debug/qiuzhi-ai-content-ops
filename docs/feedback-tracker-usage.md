# 反馈追踪工作台使用说明

## 入口与权限

公开作品集继续对外开放，运营工作台位于 `/ops`。工作台需要 ChatGPT 登录。首次初始化时，部署环境变量 `OPS_ALLOWED_EMAILS` 中的第一个邮箱成为管理员，其余邮箱成为成员；之后由管理员在“团队”页添加成员、调整角色或停用账号。

`OPS_ALLOWED_EMAILS` 只用于初始化，不应写入 Git、网页源码或截图。团队角色分为管理员、成员和只读；所有接口都会在服务端再次校验权限。

## 新增反馈

点击“新增记录”，填写平台、来源类型、匿名反馈内容、优先级、负责人和下一步。系统会自动生成 `FB-日期-随机码` 形式的追踪编号，并记录创建人和时间。

摘要中常见的手机号、邮箱和微信号会被自动遮盖，但录入前仍应先进行人工脱敏。不要粘贴私信原文、客户秘密、公司内部数据或不必要的真实昵称。

保存后可以点击“获取 AI 建议”。AI 会生成匿名摘要、情绪、需求标签和 P0–P3 优先级建议；这四项必须由成员人工确认或修改后才会更新正式字段。

## CSV 导入

在工作台下载模板，保留以下表头：

```text
平台,来源类型,平台记录编号,内容编号,用户代号,匿名反馈内容,确认摘要,情绪,需求标签,优先级,状态,负责人,下一步,发生时间
```

单次最多导入 500 条。存在“平台＋平台记录编号”重复时会跳过，其他错误会返回原 CSV 行号和安全错误说明。

## 可追踪历史

点击任意台账记录后，右侧会显示详情和 AI 建议。状态、负责人、优先级、摘要和下一步等关键变化都会产生事件记录，包括操作人和时间。多人同时编辑时，旧版本提交会被阻止并提示刷新。首版删除采用“已归档”，不做不可恢复的硬删除。

## 运营指标

运营数据区可以按平台、内容编号和日期保存指标快照，例如播放量、收藏量、评论量或关注转化。相同平台、内容、日期和指标名再次保存时会更新原快照。

## 平台自动同步

B站、抖音、小红书、视频号和公众号均有独立连接状态。没有官方开发者权限和安全授权时显示“待接入”，点击检查授权只记录一次安全的检查结果，不会抓取、回复或发布内容。

启用真实同步时，需要逐个平台提供官方 API 权限，并把令牌保存在部署密钥中。令牌不得写入 D1 或提交到仓库。

## 数据导出

“导出 CSV”会按当前筛选条件生成文件，默认不包含外部用户真实标识。导出文件不会长期保存在云端，下载后应按内部数据规范保存和清理。
