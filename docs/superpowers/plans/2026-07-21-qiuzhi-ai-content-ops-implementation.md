# 秋芝 AI 内容运营系统实施计划

## 目标

交付一个可在 Codex 中调用的内容运营 Skill、一套结构化知识库、一套完整模拟案例，以及一个可公开访问的响应式作品集网站。

## 实施原则

- 公开事实、推断和模拟数据分别标记；
- 网站不依赖登录、数据库或模型 API；
- Agent 不执行发布、评论、点赞或私信等外部写操作；
- Skill 保持精简，详细知识放入按需读取的 references；
- 每个阶段都先完成可验证的小交付，再进入下一阶段。

## 任务 1：建立项目骨架

创建以下目录和基础文件：

```text
agent-skill/qiuzhi-content-ops/
knowledge-base/
case-study/
site/
docs/
```

补充根目录 `.gitignore` 和项目说明，确保构建产物、依赖和临时文件不进入版本控制。

验收：目录清晰，Git 工作区仅出现预期文件。

## 任务 2：初始化 Codex Skill

使用官方 `init_skill.py` 在 `agent-skill/qiuzhi-content-ops` 初始化 Skill，并生成 `agents/openai.yaml`。

Skill 元数据：

- 名称：`qiuzhi-content-ops`；
- 展示名：`秋芝 AI 内容运营`；
- 触发范围：AI 博主内容策划、多平台改写、排期、用户反馈、复盘和活动节点检查；
- 默认提示：要求用户提供选题、素材或反馈，再按照知识库生成可追溯输出。

创建按需读取的 references：

- `account-profile.md`；
- `platform-playbooks.md`；
- `content-sop.md`；
- `feedback-operations.md`；
- `campaign-operations.md`；
- `output-templates.md`。

验收：Skill frontmatter 合法，openai.yaml 与 Skill 功能一致。

## 任务 3：实现 Agent 工作流

在 `SKILL.md` 中实现六类任务路由：

1. 分析选题；
2. 生成多平台内容包；
3. 制定发布排期；
4. 整理用户反馈；
5. 生成运营复盘；
6. 检查活动节点。

为每条路由定义：输入检查、需要读取的 reference、处理步骤、输出结构、异常处理和安全边界。

所有输出固定包含：主体内容、依据、待确认项、发布前检查和建议台账。

验收：对模糊选题、缺少来源和含个人信息的反馈能够安全降级处理。

## 任务 4：建立运营知识库

在 `knowledge-base` 中保留适合人类浏览的版本，内容与 Skill references 逻辑一致但服务于作品集展示。

建立：

- 账号研究档案；
- 五平台内容规范；
- 内容生产 SOP；
- 用户反馈与私信处理规范；
- 活动、社群和招聘传播节点规范；
- 发布审核清单；
- 内容日历、反馈台账、节点表和复盘模板。

验收：每项岗位职责都有对应 SOP 或模板，公开资料注明来源和查询日期。

## 任务 5：制作模拟案例

围绕“让桌面 Agent 自动整理工作记录，5 分钟生成日报和周报”制作：

- 选题评分卡；
- 资料与风险核验记录；
- B站 8—12 分钟教程策划；
- 抖音和视频号 60 秒口播及分镜；
- 小红书八页图文卡片文案；
- 公众号深度教程；
- 七天发布日历；
- 模拟评论和私信数据；
- 模拟运营数据与复盘；
- 下一期选题建议。

台账使用 JSON 或 CSV 保存，并设置明确的模拟字段。

验收：所有内容通过事实、平台适配、隐私和模拟标记检查。

## 任务 6：构建作品集网站

使用响应式前端实现七个可导航区域：

1. 首页；
2. 账号洞察；
3. Agent 工作台；
4. 知识库；
5. 完整案例；
6. 数据看板。
7. 视频速览库。

网站要求：

- 暖白背景、深色文字和黄色强调；
- 支持桌面与移动端；
- 使用真实案例文本和模拟台账数据；
- 提供可复制的现场演示指令；
- 持续显示非官方与模拟数据声明；
- 避免需要 API Key 的实时生成界面。

视频速览库使用约 30 条公开代表视频，保存标题、BV号、链接、发布日期、时长、分类、难度、适合人群、核心收获和分析说明。实现关键词、分类和难度筛选，并提供至少三条观看路线。所有视频跳转 B站原页面，不下载或重新托管视频。

验收：本地构建通过，导航、复制、筛选或标签交互正常。

## 任务 7：实现自动化验证

Agent 验证：

- 运行 `quick_validate.py`；
- 使用三个代表性提示进行前向测试；
- 检查五平台差异、来源标记、模拟标记和隐私处理。

网站验证：

- 运行 lint、测试和生产构建；
- 检查关键文案、声明和内部导航；
- 检查视频筛选、观看路线和所有 B站原视频链接；
- 渲染桌面与移动端截图进行视觉复查；
- 修复溢出、低对比度、断链和空状态问题。

内容验证：

- 扫描 `TBD`、`TODO` 和未替换占位符；
- 检查每个动态事实是否带查询日期；
- 检查每条模拟记录是否有模拟标识。

## 任务 8：安装 Skill

将验证后的 Skill 安装到个人 Codex Skills 目录，保留项目内源文件作为版本来源。

验收：个人 Skills 目录中可以发现 `qiuzhi-content-ops`，文件与项目源版本一致。

## 任务 9：发布与线上复查

将网站发布为公开链接。发布后验证：

- 首页和七个区域可以访问；
- 手机和桌面布局正常；
- 外部 B站来源链接有效；
- 网站不要求登录；
- 非官方声明和模拟数据标识可见；
- 复制演示指令功能可用。

## 任务 10：交付

提供：

- 公开网站链接；
- Agent Skill 路径；
- 项目源文件路径；
- 主要测试结果；
- 三分钟面试演示脚本；
- 后续可选增强项，但不阻塞首版交付。
