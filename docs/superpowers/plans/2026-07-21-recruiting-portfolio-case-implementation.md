# AI 知识博主内容运营投递作品实施计划

## 任务 1：内容事实底座

- 复核账号观察的公开依据和查询日期。
- 复核桌面 Agent 读取本地资料、人工确认和隐私边界相关产品能力。
- 将公开事实、内容分析、模拟设定和待核验信息分开记录。

## 任务 2：完整运营案例

- 完成一页账号观察和三个选题卡。
- 扩写主案例的 B站完整口播稿、小红书八页内容、抖音与视频号脚本、公众号提纲。
- 完成统一内容编号的七天发布排期。
- 生成 12—15 条模拟反馈，补齐去重、标签、优先级、回复判断和下一步。
- 完成目标、指标、成功标准、异常信号与下一轮实验。

## 任务 3：独立作品页

- 新增 `/portfolio` 页面和结构化内容数据。
- 实现项目摘要、选题卡、平台标签页、长稿展开、排期时间线、反馈表和复盘区。
- 增加个人项目、非官方和模拟数据标识。
- 保持现有首页与 `/ops` 路由不变。

## 任务 4：PDF作品集

- 使用与网页一致的内容制作 6—8 页 PDF。
- 检查分页、字体、表格、链接、标识和手机阅读效果。
- 将最终 PDF 放入网站下载目录并连接下载按钮。

## 任务 5：验证与发布

- 增加 `/portfolio` 内容和 PDF 下载测试。
- 运行类型检查、lint、自动化测试和生产构建。
- 检查首页、作品页、PDF和 `/ops` 登录保护。
- 提交并推送 GitHub，保存并发布新的 Sites 版本。
