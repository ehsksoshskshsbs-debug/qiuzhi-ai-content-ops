# 反馈追踪工作台实施计划

## 交付目标

在现有 Sites 项目中增加一个需要登录和邮箱白名单授权的 `/ops` 工作台，以 D1 保存反馈记录、事件历史、指标快照和平台连接状态。完成手动录入、编辑、筛选、CSV 导入导出与连接状态展示；真实平台同步在取得官方凭据前保持待接入。

## 任务 1：云端基础设施

- 将 `.openai/hosting.json` 的 D1 绑定设为 `DB`，R2 保持关闭；
- 增加数据库 schema 与初始化助手；
- 增加 ChatGPT 登录助手和服务端邮箱白名单检查；
- 为本地测试保留明确的开发模式身份，不在生产环境绕过授权。

验收：未登录请求被拒绝，非白名单用户返回 403，白名单成员可以读写 D1。

## 任务 2：反馈与审计 API

- 实现反馈列表、创建、详情和更新接口；
- 对平台、类型、优先级和状态执行枚举校验；
- 对摘要做手机号与邮箱等常见敏感信息遮盖；
- 创建和关键字段变更时写入不可由前端删除的事件记录；
- 实现汇总统计和指标快照读取。

验收：新增、编辑、筛选、去重和事件时间线测试通过。

## 任务 3：导入导出与平台连接状态

- 提供标准 CSV 模板、预检、批量导入和错误行提示；
- 根据筛选条件生成 CSV 导出；
- 建立平台连接器接口、连接状态和同步日志；
- 未配置官方授权的平台只显示“待接入”，同步接口返回可操作说明。

验收：CSV 往返不丢关键字段，重复记录不会重复写入，未授权同步不产生数据。

## 任务 4：工作台界面

- 新增 `/ops` 受保护页面；
- 实现指标卡、筛选栏、台账、详情编辑、事件时间线、CSV 操作和平台状态；
- 沿用黄、黑、米白视觉系统，桌面端使用表格，移动端使用卡片；
- 在公开首页增加工作台入口，但不暴露真实记录与统计。

验收：键盘可操作、移动端不溢出、状态不只依赖颜色表达。

## 任务 5：验证与发布

- 增加鉴权、数据校验、隐私遮盖、CSV 和渲染测试；
- 运行 lint、测试与生产构建；
- 检查公开页面不会请求受保护数据；
- 提交源码，保存新 Sites 版本并发布；
- 发布后检查公开首页、登录跳转、未授权拒绝和 D1 绑定状态。

验收：所有自动化检查通过，生产部署成功；若尚未配置成员邮箱，工作台安全地保持不可访问并明确说明下一步。

## 实施限制

- 不在源码、日志或数据库中保存平台访问令牌；
- 不自动发布、评论、点赞、回复或私信；
- 不抓取或绕过平台权限；
- 用户未提供官方 API 凭据前，不宣称已完成真实平台同步；
- 不把私信原文或可识别个人的信息带入公开网站。
