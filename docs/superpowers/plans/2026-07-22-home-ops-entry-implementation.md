# 首页团队工作台入口实施计划

## 任务 1：补充首页导航入口

- 修改 `site/app/page.tsx` 的首页导航配置，新增指向 `/ops` 的“团队工作台”。
- 为工作台链接增加独立样式标识，让受保护入口与页内锚点容易区分。
- 保留“账号洞察”和“视频速览”的现有锚点行为。

## 任务 2：补充首屏操作入口

- 在首页首屏操作区增加“打开团队工作台 · 登录后使用”按钮并指向 `/ops`。
- 保留现有视频地图和账号洞察按钮。
- 调整 `site/app/globals.css`，确保三个操作按钮在桌面端、平板和手机端清晰排列。

## 任务 3：自动化与人工验证

- 更新首页测试，断言导航和首屏都提供 `/ops` 入口。
- 确认 `/ops` 的 ChatGPT 登录保护和角色权限代码未修改。
- 运行 lint、测试和生产构建。
- 在桌面和移动视口检查入口可见性，并确认点击后进入现有登录流程。

## 任务 4：提交与发布

- 仅提交首页入口、样式、测试和相关文档。
- 推送当前分支到 GitHub。
- 通过 Sites 保存并部署新版本到现有生产网址。
- 部署成功后复核首页入口和 `/ops` 登录跳转。
