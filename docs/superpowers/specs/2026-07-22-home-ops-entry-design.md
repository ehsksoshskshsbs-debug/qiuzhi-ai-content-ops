# 首页团队工作台入口设计

## 背景

线上站点已经提供受保护的 `/ops` 团队反馈追踪工作台，但公开首页没有任何入口。访问者只能手动输入路径，因此无法从作品集页面发现数据录入、团队权限和 AI 辅助处理功能。

## 目标

让访问者在打开首页后立即知道站点包含一个可操作的团队工作台，并能清楚理解该入口需要登录。入口只负责导航，不改变既有身份验证、团队权限或数据可见性。

## 页面设计

在公开首页增加两个指向 `/ops` 的入口：

1. 主要导航新增“团队工作台”链接。
2. 首屏操作区新增醒目按钮“打开团队工作台 · 登录后使用”。

两个入口使用现有首页视觉语言，并在桌面端和移动端保持可见。现有“账号洞察”和“视频速览”入口继续保留。

## 访问流程

1. 访问者在公开首页点击“团队工作台”。
2. 浏览器进入 `/ops`。
3. 未登录用户由现有逻辑跳转至 ChatGPT 登录页。
4. 登录成功后，现有服务端权限逻辑根据团队成员和角色决定是否进入工作台。
5. 管理员、成员和只读角色继续使用现有权限范围；未授权用户继续看到现有拒绝访问页面。

## 安全与边界

- 公开首页不读取或展示任何团队反馈、评论、私信、运营数据或成员邮箱。
- 不改变 `/ops` 的 ChatGPT 登录要求。
- 不改变管理员、成员和只读角色权限。
- 不在 GitHub、浏览器页面或客户端代码中暴露 API 密钥。
- 本次不新增公开注册、自动邀请或绕过管理员审核的入口。

## 影响范围

本次只调整公开首页的导航和首屏操作区，以及必要的响应式样式。工作台页面、API、数据库结构、团队管理和 AI 分类流程不做功能修改。

## 验收标准

- 首页导航可以看到“团队工作台”。
- 首页首屏可以看到“打开团队工作台 · 登录后使用”。
- 两个入口都指向 `/ops`。
- 未登录访问 `/ops` 时进入 ChatGPT 登录流程。
- 桌面端和手机端入口均清晰可点击，不遮挡现有导航。
- 现有首页锚点、作品集内容和 `/ops` 权限逻辑不受影响。
- 项目 lint、测试和生产构建全部通过。
- 新版本部署到现有公开网址，不创建新的站点地址。

## 验证方式

1. 运行 lint、自动化测试和生产构建。
2. 在桌面视口检查导航与首屏按钮。
3. 在移动视口检查入口可见性和换行。
4. 点击两个入口，确认都进入 `/ops` 的现有登录流程。
5. 部署后复核现有生产地址。
