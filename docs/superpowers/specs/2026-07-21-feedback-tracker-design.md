# 秋芝2046 内容运营反馈追踪工作台设计

日期：2026-07-21  
状态：待用户复核  
范围：在现有公开作品集网站中增加受保护的云端运营工作台

## 1. 目标

把现有静态“模拟运营看板”升级为可长期使用的反馈追踪系统，统一整理用户反馈、评论、私信和运营数据，并为每条记录保留稳定编号、负责人、处理状态、下一步与变更历史。

系统同时支持两类数据入口：

1. 手动录入与 CSV 批量导入；
2. 平台连接器自动同步。首版完成连接器框架和连接状态管理，具体平台只有在取得官方接口权限、账号授权与密钥后才启用真实同步。

系统不得自动发布、评论、点赞、回复或发送私信。

## 2. 产品边界

### 公开区域

现有“快速了解秋芝2046”与作品集页面继续公开，保留账号研究、视频分类、Agent、知识库和案例展示。公开页面只展示汇总数字或明确标记的模拟数据，不展示运营台账中的真实内容。

### 受保护区域

新增 `/ops` 运营工作台。访问者必须通过 ChatGPT 登录，并且邮箱位于运营成员白名单中。所有读写权限均在服务端校验，不能只依赖前端隐藏按钮。

首个配置的负责人作为工作区所有者。所有获准成员共享同一工作区数据，实现多设备访问和团队协作。成员管理不在首版页面内开放，由部署环境的白名单配置控制，降低误授权风险。

## 3. 推荐架构

- 前端：沿用现有 Vinext、React 和网站视觉系统；
- 结构化持久化：Cloudflare D1，逻辑绑定名为 `DB`；
- 身份认证：Sites 提供的 Sign in with ChatGPT；
- 授权：服务端读取已认证邮箱，并与 `OPS_ALLOWED_EMAILS` 白名单核对；
- 附件存储：首版不保存截图、音频或原始导出文件，因此不启用 R2；
- CSV 导入：浏览器解析基础校验后提交标准化行，服务端再次校验并批量写入；
- 平台同步：每个平台实现独立适配器，统一输出标准化反馈与指标记录；
- 导出：服务端按当前筛选条件生成 CSV，不在云端长期保存导出文件。

公开页面与受保护工作台共用一个站点，但数据请求只能经过已鉴权的 `/api/ops/*` 接口。

## 4. 核心数据模型

### `feedback_records`

每条评论、私信或主动反馈对应一条记录：

- `id`：稳定 UUID；
- `trace_code`：便于人工沟通的编号，例如 `FB-20260721-0001`；
- `workspace_id`：所属运营工作区；
- `platform`：B站、抖音、小红书、公众号、视频号或其他；
- `source_type`：评论、私信、主动反馈、客服转交；
- `external_id`：平台侧记录编号，存在时用于去重；
- `content_id`：关联视频或图文编号；
- `author_alias`：匿名化用户代号，不保存不必要的真实昵称；
- `summary`：人工或同步后生成的匿名摘要；
- `sentiment`：正向、中性、负向；
- `need_tag`：教程需求、报错、商务、隐私、招聘、活动、其他；
- `priority`：P0、P1、P2、P3；
- `status`：待处理、处理中、等待外部、已解决、已归档；
- `assignee_email`：负责人；
- `next_action`：下一步动作；
- `occurred_at`、`created_at`、`updated_at`。

当 `platform + external_id` 均存在时建立唯一约束，防止自动同步重复入库。

### `record_events`

记录状态、负责人、优先级和下一步等关键字段的每次变化：

- 事件编号、反馈记录编号；
- 操作人邮箱；
- 动作类型；
- 变更前后值的最小 JSON 快照；
- 操作时间。

该表用于时间线展示和追责，不允许前端直接删除。

### `metric_snapshots`

按平台、内容和日期保存播放、点赞、收藏、评论、关注转化等指标。指标采用名称和值的明细行设计，便于不同平台扩展。相同平台、内容、日期与指标名保持唯一。

### `platform_connections` 与 `sync_runs`

`platform_connections` 只保存平台、连接状态、账号显示名、授权范围和最后同步时间。访问令牌只能以部署密钥或平台支持的安全凭据方式保存，不能进入 D1 明文字段、Git 或浏览器。

`sync_runs` 保存每次同步的开始时间、结束时间、成功/失败状态、新增数、更新数、跳过数和脱敏后的错误摘要。

## 5. 工作流

### 手动录入

成员填写平台、类型、内容编号和匿名摘要。系统生成追踪编号，默认状态为“待处理”，默认优先级为 P2。保存成功后生成“创建记录”事件。

### CSV 导入

下载模板 → 选择 CSV → 字段映射预览 → 本地预检 → 服务端校验 → 确认导入 → 返回成功、重复和失败行数。任何失败行都提供行号与原因，但不回显可能包含敏感信息的完整原文。

### 处理与追踪

成员可以筛选、搜索、批量指派负责人和更新状态。详情抽屉展示匿名摘要、关联内容、下一步和完整事件时间线。每次关键变更自动写入 `record_events`。

### 自动同步

连接器按“拉取游标 → 标准化 → 脱敏 → 去重 → 写入 → 记录同步结果”的固定流程工作。没有有效授权时只显示“待接入”，不得用模拟数据伪装成真实同步。

首选接入顺序为 B站、抖音、小红书，随后是视频号和公众号。实际顺序可根据用户能提供的官方接口权限调整。

## 6. 页面与交互

`/ops` 首页包括：

1. 本周新增、待处理、P0/P1、平均处理时长四个指标卡；
2. 平台、类型、优先级、状态、负责人和日期筛选；
3. 可排序的反馈台账，移动端改为记录卡片；
4. “新增记录”“导入 CSV”“导出 CSV”“平台连接”操作；
5. 反馈详情与处理时间线；
6. 平台同步状态和最近一次同步结果；
7. 运营指标趋势与内容表现摘要。

视觉延续现有黄、黑、米白作品集风格，但工作台提高信息密度，状态和优先级同时使用文字与颜色，避免只靠颜色传达。

## 7. API 轮廓

- `GET/POST /api/ops/feedback`：查询或创建记录；
- `GET/PATCH /api/ops/feedback/:id`：查看或更新记录；
- `GET /api/ops/feedback/:id/events`：读取事件时间线；
- `POST /api/ops/import/preview`：校验 CSV 映射与数据；
- `POST /api/ops/import/commit`：批量写入；
- `GET /api/ops/export`：按筛选条件导出 CSV；
- `GET/POST /api/ops/metrics`：读取或写入指标快照；
- `GET /api/ops/connections`：读取平台连接状态；
- `POST /api/ops/connections/:platform/sync`：触发一次已授权的同步。

所有接口先执行相同的身份与成员授权检查。写接口执行字段白名单、长度限制、枚举校验和参数化查询。

## 8. 隐私与安全

- 默认只保存匿名摘要，不把私信原文、手机号、微信号、公司秘密或客户信息写入系统；
- 导入前明确提示先脱敏，服务端再对常见手机号、邮箱和微信号格式做拦截或遮盖；
- 平台令牌不得进入源码、D1、日志或客户端；
- 公开页面永远不查询真实反馈明细；
- 删除功能首版采用归档，不做不可恢复的硬删除；
- 同步错误日志不保存响应正文，只保留错误类别和安全摘要；
- 导出的 CSV 按当前成员权限生成，并默认不含任何外部用户真实标识。

## 9. 失败处理

- D1 写入失败：界面保留未提交表单并显示可重试提示；
- CSV 部分失败：有效行和错误行分别计数，只有用户确认后才提交；
- 平台限流：停止当前同步并记录“受限”，不做高频重试；
- 授权过期：连接状态变为“需重新授权”，已保存台账仍可正常访问；
- 重复数据：利用平台外部编号和内容哈希跳过，并在同步结果中计数。

## 10. 分阶段交付

### 阶段一：可用的云端台账

完成认证与成员白名单、D1 数据结构、手动录入、筛选、详情、事件历史、CSV 导入导出和基础指标看板。

### 阶段二：真实平台同步

完成连接器接口、连接状态和同步日志。用户提供相应平台的官方开发者权限、授权信息与可用 API 后，逐个平台实现并测试真实拉取。

### 阶段三：运营增强

根据实际使用反馈增加批量处理、提醒、周报和更细的指标分析；任何自动回复或发布能力仍需单独授权和重新评审。

## 11. 验收标准

- 获准成员可以跨设备看到相同数据，未获准用户无法访问任何运营接口；
- 新增、编辑、指派和状态变化均生成可追踪事件；
- CSV 能预览、校验、去重、导入和导出；
- 筛选、搜索和统计结果与数据库记录一致；
- 平台未授权时明确显示待接入，不产生虚假同步数据；
- 公开作品集不会泄露受保护台账；
- lint、构建、单元测试和关键页面访问测试全部通过；
- 发布后公开页面保持可访问，运营路径要求登录并通过成员授权。

## 12. 实施前仍需的用户信息

1. 运营工作区所有者邮箱，以及需要加入白名单的成员邮箱；
2. 首个真实自动同步平台及其官方开发者权限；
3. 如果现有 CSV 有固定格式，提供一份脱敏样表用于字段映射。

在上述信息尚未提供时，阶段一仍可完成本地与部署级开发，但线上工作台会保持白名单保护，真实平台连接器显示“待接入”。
