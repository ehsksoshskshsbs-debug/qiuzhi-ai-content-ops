# 反馈与节点运营

## 用户反馈

处理顺序：匿名化 → 去重 → 情绪判断 → 需求标签 → 优先级 → 负责人角色 → 后续动作。

常用标签：安装与环境、步骤看不懂、功能限制、价格与权限、模板资料、新选题建议、事实纠错、合作商务、社群招聘、无效风险。

优先级：

- P0：安全、隐私、严重事实错误；
- P1：高频阻塞或核心步骤失败；
- P2：个别问题、资料请求或优化建议；
- P3：低相关、重复或无需动作。

## 活动、社群和招聘节点

每个节点必须包含：项目、阶段、交付物、负责人角色、截止时间、依赖项、状态、风险和证明链接。

标准阶段：Brief → 素材 → 审核 → 执行 → 复盘。

招聘传播额外检查：职责与要求一致、投递方式有效、工作信息明确、不公开候选人资料、评论问题有统一口径、到期后有停止传播节点。
